System Overview

Movie Semantic Search lets you find films by describing what you want to watch — not just by title or keyword. Type something like "a heartwarming story about a robot learning to feel" and the system surfaces the most relevant matches from thousands of movies.

The system is made up of three main parts: a data pipeline that reads and indexes movie information, a search service that understands the meaning behind your query, and a web interface where users type their searches and see results instantly.

Getting Started

  1. Start the services. Launch the system using Docker Desktop. All components — the database, the search service, and the web interface — start together with a single command from someone on the technical team.
  2. Load the movie data. Once the services are running, use the Pipeline Runner tool listed in the Operator Tools section below to load all movie information into the system. This only needs to be done once after the initial setup.
  3. Open the Search Interface. Use the Search Interface link in the Operator Tools section below to open the movie search page in your browser. From there, type any description and browse the results.

Operator Tools

These tools let you inspect, test, and operate the system. Each one is accessible directly in your browser or from the command line.

Tool Location What it does
Qdrant Dashboard http://localhost:6333/dashboard Inspect the movie database — browse what's stored and confirm the data loaded correctly.
Swagger UI http://localhost:8080/swagger-ui/index.html Try the search feature directly in the browser — type a query and see results without writing any code.
Pipeline Runner Command-line tool (run by the technical team) Loads all movie data into the system — run this once after starting the services for the first time.
Search Interface http://localhost:8080 The movie search page — the main product; type any description and find matching films.
