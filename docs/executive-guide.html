System Overview

Movie Semantic Search lets you find films by describing what you want to watch — not just by title or keyword. Type something like "a heartwarming story about a robot learning to feel" and the system surfaces the most relevant matches from thousands of movies.

The system is made up of three main parts: a data pipeline that reads and indexes movie information, a search service that understands the meaning behind your query, and a web interface where users type their searches and see results instantly.

Getting Started

  1. Export the embedding model. From the repository root, run docker compose run --rm load-model. This exports the ONNX model into the Triton model repository and must complete before the services start.
  2. Start the services. Launch the system using Docker Desktop by running docker compose up -d. All components — the database, the search service, and the web interface — start together.
  3. Load the movie data. From the repository root, run docker compose run --rm load-data to load all movie information into the system. This only needs to be done once after the initial setup.
  4. Open the Search Interface. Use the Search Interface link in the Operator Tools section below to open the movie search page in your browser.

Operator Tools

These tools let you inspect, test, and operate the system. Each one is accessible directly in your browser or from the command line.

Tool Location What it does
Qdrant Dashboard http://localhost:6333/dashboard Inspect the movie database — browse what's stored and confirm the data loaded correctly.
Swagger UI http://localhost:8080/swagger-ui/index.html Try the search feature directly in the browser — type a query and see results without writing any code.
Pipeline Runner Command-line tool (run by the technical team) Loads all movie data into the system — run this once after starting the services for the first time.
Search Interface http://localhost:8080 The movie search page — the main product; type any description and find matching films.

End-to-End Test Walkthrough

Prerequisites

The repository root is the folder containing Makefile, docker-compose.yml, api/, pipeline/, and docs/.

  1. Get a TMDB API key. Create a free account at https://www.themoviedb.org/signup. After logging in, navigate to Settings → API and generate an API key. Copy the key — you will use it in the next step.
  2. Configure your environment file. From the repository root, run the following command, replacing paste_your_key_here with your actual TMDB API key: 
    TMDB_KEY=paste_your_key_here && cp .env.example .env && sed -i "s|TMDB_API_KEY=.*|TMDB_API_KEY=$TMDB_KEY|; s|PROJECT_ROOT=.*|PROJECT_ROOT=$(pwd)|" .env
  3. Install Docker Desktop. Download and install Docker Desktop from https://www.docker.com/products/docker-desktop/. Ensure Docker Desktop is running before continuing.
  4. Export the embedding model. From the repository root, run the following command. This must complete before starting the services: 
    docker compose run --rm load-model
  5. Start all services. From the repository root, run the following command and wait approximately 30 seconds for all containers to become healthy: 
    docker compose up -d
  6. Load the movie data. From the repository root, run the following command. This may take several minutes: 
    docker compose run --rm load-data
  7. Confirm the system is ready. Open http://localhost:6333/dashboard in your browser. If you see the Qdrant Dashboard without errors, all services are running and you are ready to begin the walkthrough. If you see a connection error, wait 30 seconds and refresh.

Walkthrough Steps

  1. Verify the Qdrant collection record count. How to verify: Open http://localhost:6333/dashboard, click the movies collection, and note the number shown under "Points count." Expected outcome: The movies collection shows a non-zero record count (typically several thousand records), confirming that the data pipeline loaded movie data into the vector database.
    If this doesn't work: The record count shows 0 or the collection does not exist. From the repository root, re-run the data loader: 
    docker compose run --rm load-data
    If the collection still shows 0 records after the loader completes, file a bug using /create-feature-request: 
    Title: Qdrant movies collection empty after load-data run

Actual: Points count = 0

Expected outcome: Each record contains a populated embedding — a long array of floating-point numbers — confirming that text embedding was applied to each movie description during ingestion.
