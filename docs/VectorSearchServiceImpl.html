VectorSearchServiceImpl

Real implementation of VectorSearchService that calls Qdrant's REST API to retrieve the top-N movies by cosine similarity to a query embedding vector.

Interface Contract

public interface VectorSearchService { VectorSearchResponse search(VectorSearchRequest request); } // VectorSearchRequest wraps float[] vector + int limit // VectorSearchResponse wraps List<MovieResult>

Request Marshalling

The implementation builds a QdrantSearchRequest with the query vector, the requested limit, and with_payload = true. This is POST-ed to {qdrant.base-url}/collections/movies/points/search. Jackson serialises the float array directly into the Qdrant JSON schema.

Response Mapping

Each element in the result array is deserialized into a QdrantResult containing a cosine score and a payload. The payload fields are mapped to MovieResult via a stream:

Error Handling

ConditionSpring exceptionThrown as
Qdrant unreachable / connection refused ResourceAccessException VectorSearchServiceException("…connection refused")
Qdrant returns non-2xx HTTP status HttpStatusCodeException VectorSearchServiceException("…error response")

VectorSearchServiceException is a RuntimeException. GlobalExceptionHandler maps it to HTTP 503.

Implementation Sketch

@Service @ConditionalOnProperty(name = "qdrant.mock", havingValue = "false", matchIfMissing = false) public class VectorSearchServiceImpl implements VectorSearchService { private final RestTemplate restTemplate; private final String searchUrl; public VectorSearchServiceImpl(RestTemplate restTemplate, QdrantProperties properties) { this.restTemplate = restTemplate; this.searchUrl = properties.getBaseUrl() + "/collections/movies/points/search"; } @Override public VectorSearchResponse search(VectorSearchRequest request) { QdrantSearchRequest body = new QdrantSearchRequest(request.getVector(), request.getLimit()); try { ResponseEntity<QdrantSearchResponse> resp = restTemplate.postForEntity(searchUrl, body, QdrantSearchResponse.class); List<MovieResult> results = resp.getBody().result.stream() .map(r -> MovieResult.builder() .title(r.payload.title) .year(r.payload.releaseYear) .genres(r.payload.genres) .score(r.score) .summarySnippet(r.payload.summarySnippet) .thumbnailUrl(r.payload.thumbnailUrl) .build()) .toList(); return new VectorSearchResponse(results); } catch (ResourceAccessException e) { throw new VectorSearchServiceException(CONNECTION_REFUSED, e); } catch (HttpStatusCodeException e) { throw new VectorSearchServiceException(NON_2XX_RESPONSE); } } }

Mock vs Real Wiring

When qdrant.mock=true (or the property is absent) in application.yml, Spring wires MockVectorSearchService instead, which returns a fixed list of five movies without contacting Qdrant. This allows full search-flow testing in environments without a running Qdrant container.
