TritonGrpcConfig

Spring @Configuration class that creates and manages the gRPC ManagedChannel to Triton Inference Server and exposes the generated blocking stub as a Spring bean.

flowchart LR
  PROPS["TritonProperties\nhost / port / deadlineMs"]
  CONFIG["TritonGrpcConfig\n@Configuration"]
  CHANNEL["ManagedChannel\nNettyChannelBuilder"]
  STUB["GRPCInferenceServiceBlockingStub"]
  EMBED["EmbeddingServiceImpl\n@Service"]

  PROPS --> CONFIG
  CONFIG --> CHANNEL
  CHANNEL --> STUB
  STUB --> EMBED

Responsibilities

Creates a single shared ManagedChannel backed by Netty, pointed at the Triton gRPC endpoint configured via TritonProperties. Exposes a GRPCInferenceServiceBlockingStub bean wired with a per-call deadline. Registers a @PreDestroy hook that shuts the channel down gracefully on application stop.

Configuration Properties (application.yml)

KeyDefaultDescription
triton.hostlocalhostHostname or IP of the Triton Inference Server.
triton.port8001gRPC port on the Triton server.
triton.model-nameall-MiniLM-L6-v2Model name used in ModelInferRequest.model_name.
triton.deadline-ms5000Per-call deadline in milliseconds applied to every gRPC request.
triton.mockfalseWhen true, Spring wires MockEmbeddingService instead of the real gRPC implementation.

Bean Wiring Sketch

@Configuration @EnableConfigurationProperties(TritonProperties.class) public class TritonGrpcConfig { private final TritonProperties props; private ManagedChannel channel; @Bean public GRPCInferenceServiceBlockingStub tritonStub() { channel = NettyChannelBuilder .forAddress(props.getHost(), props.getPort()) .usePlaintext() .build(); return GRPCInferenceServiceGrpc.newBlockingStub(channel); } @PreDestroy public void shutdown() throws InterruptedException { if (channel != null) { channel.shutdown().awaitTermination(5, TimeUnit.SECONDS); } } }

Design Notes

The channel is plaintext (usePlaintext()) because Triton is accessed within the same Docker network — TLS termination is not required. The stub is blocking (synchronous) to keep the Spring MVC thread model simple; async stubs would require reactive wiring not currently present in the API. The triton.mock flag allows the entire gRPC stack to be bypassed in local development or CI without a running Triton container.
