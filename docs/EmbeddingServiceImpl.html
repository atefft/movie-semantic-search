EmbeddingServiceImpl

Real implementation of EmbeddingService that calls Triton Inference Server over gRPC to convert a query string into a 384-dimensional float32 embedding vector.

Interface Contract

public interface EmbeddingService { EmbeddingResponse embed(EmbeddingRequest request); } // EmbeddingRequest wraps the query string // EmbeddingResponse wraps float[] vector (384 dims)

Request Marshalling

The implementation packs the raw text string into a ModelInferRequest targeting the all-MiniLM-L6-v2 model. The TEXT input tensor has data_type = TYPE_BYTES and shape = [1, 1]. The text bytes are written to the request's raw_input_contents field, length-prefixed as required by the Triton binary protocol.

Response Parsing

The response contains a sentence_embedding output tensor with data_type = TYPE_FP32 and shape = [1, 384]. The raw bytes are read from raw_output_contents, then interpreted as 384 little-endian float32 values using a ByteBuffer with LITTLE_ENDIAN order.

Error Handling

ConditionThrownHTTP mapping
Triton unreachable / connection refusedEmbeddingServiceException503 Service Unavailable
gRPC deadline exceeded (default 5 s)EmbeddingServiceException503 Service Unavailable
Triton model not loaded / unknown modelEmbeddingServiceException503 Service Unavailable

All StatusRuntimeExceptions from the gRPC stub are caught and re-thrown as EmbeddingServiceException, which GlobalExceptionHandler maps to HTTP 503.

Implementation Sketch

@Service @ConditionalOnProperty(name = "triton.mock", havingValue = "false", matchIfMissing = true) public class EmbeddingServiceImpl implements EmbeddingService { private final GRPCInferenceServiceBlockingStub stub; private final TritonProperties props; @Override public EmbeddingResponse embed(EmbeddingRequest request) { try { ModelInferRequest inferRequest = buildRequest(request.getText()); ModelInferResponse response = stub .withDeadlineAfter(props.getDeadlineMs(), TimeUnit.MILLISECONDS) .modelInfer(inferRequest); float[] vector = parseEmbedding(response); return new EmbeddingResponse(vector); } catch (StatusRuntimeException e) { throw new EmbeddingServiceException("Triton gRPC call failed", e); } } }

Mock vs Real Wiring

When triton.mock=true in application.yml, Spring wires MockEmbeddingService instead, which returns a deterministic alternating [0.1, -0.1, ...] vector without contacting Triton. This allows full search-flow testing in environments without a running Triton container.
