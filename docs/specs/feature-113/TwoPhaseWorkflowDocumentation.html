TwoPhaseWorkflowDocumentation

#113 — Update documentation for the two-phase pipeline workflow

Spec for documentation updates across 7 files — replacing all references to the old single-phase pipeline command with the new three-step workflow.

Overview

All user-facing and developer-facing documentation must reflect the two-phase pipeline execution model: Phase 1 (load-model) runs before services start and exports the ONNX model; Phase 2 (load-data) runs after services are healthy and loads all movie data. Every existing reference to docker compose run --rm pipeline and make pipeline (as a one-shot runner) must be replaced with the correct three-step sequence.

The old documentation had a sequencing bug: Triton was started before the ONNX model was exported, causing Triton to fail to load the model. The new workflow fixes this by running load-model before docker compose up -d.

The authoritative three-step sequence that must appear in all updated documentation:

# Step 1 — export model (before services start) docker compose run --rm load-model # Step 2 — start services docker compose up -d # Step 3 — load data (after services are healthy) docker compose run --rm load-data
