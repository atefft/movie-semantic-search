DockerComposeRunner

Service interface and implementation for running docker compose subcommands from project.root. Used by TritonServiceManagerImpl and QdrantServiceManagerImpl to start and stop services.

Interface

public interface DockerComposeRunner { void run(String... args) throws IOException, InterruptedException; }

Implementation: DockerComposeRunnerImpl

Builds a docker compose <args> command and launches it via ProcessBuilder with project.root as the working directory. Throws IOException if the process exits with a non-zero code.

@Service public class DockerComposeRunnerImpl implements DockerComposeRunner { private final ProjectProperties projectProperties; @Override public void run(String... args) throws IOException, InterruptedException { List<String> cmd = new ArrayList<>(); cmd.add("docker"); cmd.add("compose"); for (String arg : args) cmd.add(arg); Process process = processLauncher.launch(cmd, projectProperties.getRoot()); int exitCode = process.waitFor(); if (exitCode != 0) throw new IOException("docker compose exited with code: " + exitCode); } }

Behavior

Caller invocationShell commandWorking dir
run("up", "-d", "triton") docker compose up -d triton project.root
run("stop", "triton") docker compose stop triton project.root
run("up", "-d", "qdrant") docker compose up -d qdrant project.root
run("stop", "qdrant") docker compose stop qdrant project.root

Error Handling

A non-zero exit code is converted to IOException. Callers (TritonServiceManagerImpl, QdrantServiceManagerImpl) catch this and wrap it in a domain-specific exception (TritonServiceException, QdrantServiceException). InterruptedException is propagated up and the thread interrupt flag is restored.
