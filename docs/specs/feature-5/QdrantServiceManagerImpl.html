QdrantServiceManagerImpl

Real implementation of QdrantServiceManager. Starts and stops the Qdrant container via DockerComposeRunner and polls GET <qdrant.baseUrl>/health for live status.

Operations

MethodDocker Compose commandError type
start() docker compose up -d qdrant QdrantServiceException (START_FAILED)
stop() docker compose stop qdrant QdrantServiceException (STOP_FAILED)
getStatus() HTTP GET <qdrant.baseUrl>/health Returns STOPPED on any exception

Health Check

getStatus() issues GET <qdrant.baseUrl>/health (e.g. http://localhost:6333/health). HTTP 200 → RUNNING; any other response or exception → STOPPED. Note: the health endpoint is /health not /healthz — the Docker Compose healthcheck uses /healthz but the Java client calls /health.
