TritonServiceManagerImpl

Real implementation of TritonServiceManager. Starts and stops the Triton container via DockerComposeRunner and polls GET :8000/v2/health/ready for live status.

Operations

MethodDocker Compose commandError type
start() docker compose up -d triton TritonServiceException (START_FAILED)
stop() docker compose stop triton TritonServiceException (STOP_FAILED)
getStatus() HTTP GET :8000/v2/health/ready Returns STOPPED on any exception

Health Check

getStatus() constructs the URL as http://<triton.host>:8000/v2/health/ready and issues a GET with a 2-second connect timeout. HTTP 200 → RUNNING; any other response or exception → STOPPED. Status is always polled live — never cached.
