#88 — Executive guide: end-to-end testing walkthrough

Spec for the End-to-End Test Walkthrough section of docs/executive-guide.html — a step-by-step guide for non-technical operators to verify the full Movie Semantic Search pipeline from setup through a live search query.

Overview

The WalkthroughSection expands the stub id="walkthrough" section in the executive guide with a complete, operator-facing end-to-end test procedure. It covers environment setup (TMDB API key, .env configuration, Docker Desktop installation), service startup, data pipeline execution, and five verification steps that confirm the system is functioning correctly. Each verification step includes an expected outcome and a failure remediation block so an operator can resolve issues without engineering support.

Pipeline Steps

Prerequisites

  1. Get a TMDB API key. Create a free account at https://www.themoviedb.org/signup. After logging in, navigate to Settings → API and generate an API key.
  2. Configure your environment file. From the repository root, run the following one-liner, replacing paste_your_key_here with your actual TMDB API key:
    TMDB_KEY=paste_your_key_here && cp .env.example .env && sed -i "s|TMDB_API_KEY=.*|TMDB_API_KEY=$TMDB_KEY|; s|PROJECT_ROOT=.*|PROJECT_ROOT=$(pwd)|" .env
  3. Install Docker Desktop. Download from https://www.docker.com/products/docker-desktop/. Ensure Docker Desktop is running before continuing.
  4. Start all services. From the repository root, run:
    docker compose up -d
    Wait approximately 30 seconds for all containers to become healthy.
  5. Load the movie data. From the repository root, run the data pipeline:
    docker compose run --rm load-data
    This may take several minutes.
  6. Confirm the system is ready. Open http://localhost:6333/dashboard in your browser. A visible Qdrant Dashboard without errors confirms all services are running.

Failure Remediation Blocks

Each walkthrough step includes a failure remediation block with a recovery command and a bug-report template. The remediation blocks follow a consistent pattern:

Step 1 — Qdrant collection empty (Points count = 0)

Re-run the pipeline from the repository root:

docker compose run --rm load-data

If the collection still shows 0 records after the pipeline completes, file a bug using /create-feature-request with title: Qdrant movies collection empty after pipeline run.

Step 2 — Vector field null, empty, or absent on stored records

Re-run the pipeline to regenerate embeddings:

docker compose run --rm load-data

If vectors remain missing, file a bug: Qdrant movie records missing vector embeddings.

Step 3 — Collection status not Green (Red, Yellow, or "Indexing")

Wait 60 seconds and refresh. If the problem persists, restart Qdrant:

docker compose restart qdrant

If the status never becomes Green, file a bug: Qdrant movies collection stuck in non-Green status.

Step 4 — GET /api/search returns error or empty results

Confirm the API service is running:

docker compose ps api

If the container is not running, start it:

docker compose up -d api

If the API is running but returns errors, file a bug: GET /api/search returns error or empty results for "sci-fi space adventure".

Step 5 — Search results are not relevant

Re-run the pipeline to verify embeddings were generated correctly:

docker compose run --rm load-data

If results remain irrelevant after re-running, file a bug: Search results for "sci-fi space adventure" are not relevant.
