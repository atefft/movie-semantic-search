SwaggerUiConfig

Swagger UI / OpenAPI 3 explorer for the Spring Boot API — added via springdoc-openapi-starter-webmvc-ui with zero additional configuration.

Dependency

<!-- api/pom.xml --> <dependency> <groupId>org.springdoc</groupId> <artifactId>springdoc-openapi-starter-webmvc-ui</artifactId> <version>2.3.0</version> </dependency>

No @Configuration class is needed. Adding the dependency to pom.xml is sufficient — Spring Boot auto-configuration registers the Swagger UI servlet and the OpenAPI JSON endpoint automatically.

Endpoints

PathDescription
http://localhost:8080/swagger-ui/index.html Interactive Swagger UI — browse and try all documented API endpoints in a browser
http://localhost:8080/v3/api-docs Raw OpenAPI 3 JSON schema — consumed by Swagger UI and compatible with external tooling

Documented API Endpoints

MethodPathControllerDescription
GET /api/search SearchController Semantic movie search. Params: q (required), limit (1–50, default 10)
GET /api/operator/health OperatorController Returns Triton, Qdrant, and API health status as booleans
GET /api/operator/status OperatorController Returns current pipeline step statuses
GET /api/operator/run/{step} OperatorController Runs a numbered pipeline step; streams output via SSE
GET /api/operator/run/all OperatorController Runs all pipeline steps; streams output via SSE. Param: force (boolean)
POST /api/operator/service/{name}/start OperatorController Starts a named service (triton or qdrant)
POST /api/operator/service/{name}/stop OperatorController Stops a named service (triton or qdrant)

Configuration Notes

springdoc-openapi 2.x targets Spring Boot 3 / Spring Framework 6. It scans @RestController classes at startup and derives the OpenAPI schema from method signatures, parameter annotations, and return types — no manual schema authoring required. The UI is served as static resources bundled in the starter JAR.
