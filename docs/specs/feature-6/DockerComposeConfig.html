DockerComposeConfig

Documents docker-compose.yml at the repo root — defines the three services (Triton, Qdrant, API) and their wiring.

Service Details

triton

Mounts ./model-repository:/models. Exposes HTTP (8000), gRPC (8001), and metrics (8002) ports. The healthcheck polls GET :8000/v2/health/ready to confirm the model is loaded before dependent services start.

qdrant

Uses named volume qdrant_data:/qdrant/storage to persist the vector index across restarts. Exposes REST (6333) and gRPC (6334) ports. The healthcheck polls GET :6333/healthz.

api

Built from the repo-root Dockerfile. Exposed on port 8080. Declares depends_on for both triton and qdrant with condition: service_healthy, ensuring the API container only starts after both backends pass their healthchecks.

Dependency Ordering

Docker Compose starts triton and qdrant in parallel, waits for both to report healthy, then starts api. This guarantees that the Spring Boot application can connect to Triton on gRPC port 8001 and Qdrant on REST port 6333 immediately on startup.

Named Volumes

volumes: qdrant_data: {}

The qdrant_data volume is declared at the top level so Docker manages its lifecycle independently of the container. It is removed only when make clean runs docker compose down -v.
