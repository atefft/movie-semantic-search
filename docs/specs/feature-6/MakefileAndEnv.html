MakefileAndEnv

Documents Makefile and .env.example at the repo root — convenience targets for managing the Docker Compose stack and the data pipeline.

Makefile Targets

TargetCommand(s)Description
up docker compose up -d Start all services in detached mode.
down docker compose down Stop and remove containers (volumes retained).
pipeline docker compose run --rm load-model, then docker compose up -d, then docker compose run --rm load-data Orchestrate the two-phase data loading workflow: export the ONNX model, start all services, then load all movie data. Exits with a labelled error message if any phase fails.
clean Remove model artefacts and data dirs, then docker compose down -v Full teardown: wipe generated files and destroy named volumes.

Pipeline Target Detail

The pipeline target orchestrates a three-phase workflow: first it runs docker compose run --rm load-model to export the ONNX model, then docker compose up -d to start all services, then docker compose run --rm load-data to fetch and embed movie data into Qdrant. Each phase must succeed before the next begins; a failure exits with a labelled error message identifying which phase failed.

Clean Target Detail

The clean target removes generated model artefacts from model-repository/all-minilm-l6-v2/1/ and all pipeline data directories, then runs docker compose down -v to destroy the qdrant_data named volume. Use this to reset the entire environment to a clean state.

Environment Variables (.env.example)

VariableDescriptionRequired
TMDB_API_KEY API key for The Movie Database; used by pipeline/05_enrich_tmdb.py to fetch movie metadata. Yes
PROJECT_ROOT Absolute path to the repo root — present in .env.example for historical reasons; not used by any pipeline script. No

Copy .env.example to .env and set TMDB_API_KEY before running make pipeline. The .env file is gitignored to prevent accidental credential commits.
