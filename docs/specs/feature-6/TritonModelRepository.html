TritonModelRepository

Documents model-repository/all-minilm-l6-v2/config.pbtxt — the Triton Inference Server configuration for the sentence-embedding model.

Model Config (config.pbtxt)

name: "all-minilm-l6-v2" backend: "python" input [ { name: "TEXT", data_type: TYPE_BYTES, dims: [1] } ] output [ { name: "sentence_embedding", data_type: TYPE_FP32, dims: [384] } ] dynamic_batching { preferred_batch_size: [8, 16] max_queue_delay_microseconds: 5000 }

Input / Output Tensors

DirectionNameData TypeDimsDescription
input TEXT TYPE_BYTES [1] Raw UTF-8 text string to embed, length-prefixed per Triton binary protocol.
output sentence_embedding TYPE_FP32 [384] 384-dimensional float32 embedding vector produced by all-MiniLM-L6-v2.

Dynamic Batching

Triton batches concurrent inference requests before forwarding them to the Python backend. Preferred batch sizes are 8 and 16. If a full batch is not available within 5 ms (max_queue_delay_microseconds: 5000), the partial batch is dispatched immediately. This keeps tail latency bounded while improving GPU/CPU throughput under load.

Model Directory Layout

Triton requires a versioned subdirectory under the model name:

model-repository/ all-minilm-l6-v2/ config.pbtxt # model configuration (committed) 1/ # version directory model.onnx # generated by pipeline/02_export_onnx.py (gitignored) model.py # Python backend entry point (committed)

model-repository/all-minilm-l6-v2/1/model.onnx is generated by pipeline step 02 and is gitignored. Run docker compose run --rm load-model to produce it before starting the services. Alternatively, run python pipeline/02_export_model.py directly.
